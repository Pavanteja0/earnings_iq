# Security Policy

We take security and corporate confidentiality seriously. Because EarningsIQ parses sensitive financial documents, safeguarding credentials, access tokens, and documents is a top priority.

---

## Supported Versions

Only the latest release on the `main` branch is actively supported with security updates.

| Version | Supported |
| :--- | :--- |
| `v1.1.x` (current) | ✅ Yes |
| `< v1.1.0` | ❌ No |

---

## 🔒 Reporting a Vulnerability

**Please do not open a public GitHub Issue for security vulnerabilities.**

Instead, report vulnerabilities responsibly by:
1. **Emailing the Maintainers**: Send a message to `security@earningsiq.example.com` (or submit a report via GitHub's Private Vulnerability Reporting panel).
2. **Details to Include**:
   * Description of the vulnerability or risk (e.g. API credential leakage, ChromaDB access bypass).
   * Steps to reproduce the issue (including sample input files if relevant).
   * Potential impact details.

We will acknowledge receipt within 48 hours and provide a fix or mitigation plan within 7 days.
